3Configure kubeconfig

🪟 Windows Server

# Create the .kube directory
New-Item -ItemType Directory "$env:USERPROFILE\.kube" -Force

# Copy kubeconfig from cluster (WinSCP / shared folder / USB)
# Destination:
$env:USERPROFILE\.kube\config

# If IP is 127.0.0.1 (common with k3s), replace with node IP:
(Get-Content "$env:USERPROFILE\.kube\config") `
  -replace '127\.0\.0\.1', '192.168.X.X' `
  | Set-Content "$env:USERPROFILE\.kube\config"

kubectl cluster-info   # verify

🐧 Linux

# k3s
scp user@<node-ip>:/etc/rancher/k3s/k3s.yaml ~/.kube/config

# kubeadm
scp user@<node-ip>:/etc/kubernetes/admin.conf ~/.kube/config

# RKE2
scp user@<node-ip>:/etc/rancher/rke2/rke2.yaml ~/.kube/config

# Secure the file and verify
chmod 600 ~/.kube/config
kubectl cluster-info
⚠️
The kubeconfig file contains cluster admin credentials. Never commit it to git. Treat it like a password file.

4Run the Deploy Script

🪟 Windows Server — deploy.ps1

cd arrs-kube-plex-nest

# Preview (no changes):
.\deploy.ps1 -DryRun

# Deploy:
.\deploy.ps1

# Custom namespace or values override:
.\deploy.ps1 -Namespace media -ValuesFile my-values.yaml

🐧 Linux — deploy.sh

cd arrs-kube-plex-nest

# Preview (no changes):
./deploy.sh --dry-run

# Deploy:
./deploy.sh

# Custom namespace or values override:
./deploy.sh -n media -f my-values.yaml

What the deploy script prompts for

NAS Host
IP or hostname
Your NAS address — never stored in files
SMB Share
Share name
Share name on the NAS (default: media) + quota size
SMB Credentials
Username + Password
SMB user — stored as Kubernetes Secret
Plex Claim Token
plex.tv/claim
Valid 4 min — get it right before running
VPN Credentials
Username + Password
Stored as openvpn.cred Secret (0400)
ClamAV Webhook
ntfy / Discord / Slack URL
Leave blank to disable alerts
🔒
All password prompts are hidden (Read-Host -AsSecureString / read -s). Values go directly into kubectl create secret and are never written to disk.

5Post-Deploy Checklist

  • ☐
    Upload .ovpn file via filebrowser
    kubectl port-forward svc/sabnzbd-filebrowser 8888:8888 -n media → http://localhost:8888 (admin/admin — change password!)
  • ☐
    Restart SABnzbd to apply patched .ovpn
    kubectl rollout restart deployment/sabnzbd -n media
  • ☐
    Claim Plex server (plex-0)
    Visit http://<plex-0-LB-IP>:32400/web · claim token entered during deploy · fresh token: plex.tv/claim
  • ☐
    Connect Sonarr + Radarr to SABnzbd
    Host: sabnzbd.media.svc.cluster.local · Port: 8080 · API key: SABnzbd → Settings → General
  • ☐
    Verify VPN is active
    kubectl port-forward svc/sabnzbd-vpn 8000:8000 -n media → curl http://localhost:8000/v1/openvpn/status
  • ☐
    Confirm Plex HPA is getting CPU metrics
    kubectl get hpa plex-hpa -n media (TARGETS column should show a % not "<unknown>")
  • ☐
    Run a test ClamAV scan
    kubectl create job --from=cronjob/clamav-daily test-scan -n media
